# PRD — SpaceKkabbi 공식 홈 (선생님 버전)

> 원본: 학생B의 sooktday-bot/first-commit fork · 이 문서가 모든 작업의 기준이다.
> 까비 생태계 전체 그림은 first-teacher의 [까비 월드 비전 문서](https://github.com/SeoBoSeok/first-teacher/blob/main/docs/05-vision-kkabbi-world.md) 참고.

## 1. 제품 비전

**SpaceKkabbi(까비 NFT IP)의 공식 홈** — 세계관(팩션·캐릭터)을 소개하고, 팬이 회원이 되고,
굿즈/멤버십을 결제하고, 까비 월드(학생A의 게임)로 들어가는 **관문**이 되는 웹 서비스.

까비 생태계에서의 역할 분담:
- **까비 월드(학생A)**: 팬들이 모여 노는 "본진" (실시간 멀티 게임)
- **공식 홈(학생B, 이 프로젝트)**: 세계관·멤버십·결제·공지 — IP의 "공식 창구"
- 최종적으로 홈 안에 월드가 임베드된다 (M4)

## 2. 현재 상태 (As-Is)

✅ 이미 잘 만들어진 것 (초보 수준을 한참 넘었다):
- 회원 가입·로그인 (이메일+비밀번호 bcrypt 해시, 이메일 인증 Resend, Discord OAuth)
- Stripe 결제 흐름 (checkout → success/cancel → 웹훅 수신)
- 닉네임 설정, 계정 삭제(DANGER ZONE), 주문 조회 API
- 세계관 데이터 구조화 (`spacekkabbi-data.ts` — 팩션 3개, 캐릭터, 아티팩트)
- 음악 플레이어, R3F 3D 요소
- **보안 기본기**: `.env`·`data/` git 제외, 비밀번호 해시 — 습관이 잡혀 있음 👏

❌ 완성까지의 벽:
- **데이터가 파일(JSON)에 저장됨** — `store.ts` 주석에 "Prisma 임시 대체"라고 스스로 적어둠.
  Vercel은 파일 쓰기가 안 되므로 **이대로는 배포 불가** (이 수업의 LMS가 겪은 것과 같은 벽)
- `SpacekkabbiApp.tsx` 한 파일 934줄 — 수정 난도 상승 중
- 결제의 실패·취소 경로와 웹훅 서명 검증 마무리 필요
- 아직 localhost에만 존재 — 배포 안 됨

## 3. 마일스톤 (완성 정의)

| 단계 | 목표 | 완료 기준 (DoD) |
|------|------|----------------|
| **M1. 진짜 데이터베이스** | 파일 스토어 → DB | `store.ts`의 함수 표면은 유지한 채 내부를 Prisma+Postgres로 교체. 서버 재시작해도 회원이 남아 있음 |
| **M2. 회원·결제 완성** | 끝까지 가는 흐름 | 가입→인증→로그인→결제→주문확인이 테스트 카드로 E2E 통과. 실패·취소 화면, 웹훅 서명 검증 |
| **M3. 세상에 공개** | Vercel 배포 | 공개 URL에서 전 기능 동작 (환경변수는 Vercel에). 친구 1명 가입 성공 |
| **M4. 까비 생태계 연결** | 월드 임베드 + NFT | `/world` 페이지에 까비 월드 iframe(닉네임 postMessage 전달), OpenSea SpaceKkabbi 갤러리 페이지 |

## 4. 비기능 요구사항

- **보안 최우선**: 비밀키는 환경변수로만, 웹훅은 서명 검증, 모든 입력은 zod 검증
- **결제는 끝까지 테스트 모드**: 실결제 전환은 수업 범위 밖 (사업자·세금 문제)
- Next.js 16 규칙 준수 — `AGENTS.md` 안내대로 동봉 문서 우선 참조
- 모바일 브라우저에서 주요 흐름 동작

## 5. 하지 않을 것 (Out of Scope)

- 실결제 전환, 정산·세금 처리
- 어드민 대시보드 (주문 확인은 당분간 DB 콘솔로)
- 자체 채팅·커뮤니티 기능 — 그건 까비 월드(학생A) 영역. 중복 개발 금지
- 마이크로서비스·모노레포 등 구조 욕심 — 지금 구조로 충분
