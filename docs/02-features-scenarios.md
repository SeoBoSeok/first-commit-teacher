## 2. 시나리오 (유저 스토리)

### S1. 팬의 첫 방문 (해피 패스)
1. 링크 접속 → 세계관(팩션·캐릭터)을 구경하고 음악을 듣는다.
2. 마음에 들어 가입 → 인증 메일 클릭 → 로그인 → 닉네임 설정.
3. 멤버십/굿즈를 테스트 카드(4242…)로 결제 → 주문 확인.
4. `/world`로 까비 월드 입장 — 닉네임 그대로 친구들과 만난다 (M4).

### S2. 결제가 틀어졌을 때 (가장 중요한 실패 시나리오 — M2)
1. 결제 중 카드 거절(테스트 카드 4000…0002) → **무슨 일이 있었는지** 설명과 재시도 버튼.
2. 결제창에서 뒤로가기/취소 → cancel 페이지가 장바구니를 잃지 않는다.
3. 웹훅이 두 번 와도 주문은 한 건만 생성된다.

### S3. 인증 메일이 안 왔을 때 (M2)
1. 재발송 버튼 → 새 토큰 발급, 이전 토큰 무효화.
2. 만료된 링크 클릭 → "만료됐어요, 다시 보내드릴까요?" (500 에러 ❌)

### S4. 떠나는 회원
1. DANGER ZONE에서 삭제 → 확인 절차 → 세션 종료, 재로그인 불가.

### S5. 악의적 방문자 (보안 시나리오)
- 남의 주문 ID 조회 시도 → 403. 웹훅 위조 POST → 서명 불일치로 거부.
- 가입 폼에 스크립트 주입 → zod 검증과 React 이스케이프로 무력화.
